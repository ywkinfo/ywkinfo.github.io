# Hola Diario 웹앱 정확판 리뷰 보고서

> 이 문서는 현재 코드 상태와 실행 우선순위를 맞추기 위한 내부 실행안입니다. 시각 자료나 장식성 제안보다 사실 동기화와 위험도 정렬을 우선합니다.

## 검토 기준

- 코드 확인 범위: 라우트, 저장 구조, 세션 복원, Gemini 프록시, 네비게이션, 복습 플로우, 테스트 범위
- 재검증 결과: `npm run lint`, `npm run test -- --run`, `npm run build` 통과
- 문서 목적: 현재 구현 설명과 실행 우선순위 정리

## 현재 확인된 상태

- 앱 라우트는 `/`, `/add`, `/deck`, `/review`, `/summary`, `/settings`로 구성됩니다.
- 카드 데이터와 복습 상태는 IndexedDB에 저장됩니다. 저장소는 `entries`와 `reviews` 두 스토어를 사용합니다.
- 진행 중인 복습 세션과 마지막 세션 요약은 `localStorage`에 별도로 저장되며, 새로고침 후 복원됩니다.
- 현재 `StudyRepository`는 `createEntry`, `updateEntry`, `listEntries`, `getDueEntries`, `recordReview`, `getStats`, `exportData`를 제공합니다.
- Gemini 자동 채우기는 Vite 플러그인 프록시를 통해 `/api/spanish-word-autofill`로 제공되며, API 키는 브라우저에 직접 노출되지 않습니다.
- 홈 복귀는 하단 탭이 아니라 상단 브랜드 링크를 통해 가능합니다. 하단 네비게이션은 `단어장`, `추가`, `복습`, `요약`, `설정` 5개 탭입니다.
- 현재 테스트는 `srs`, `validation`, `wordAssist`, `App` 통합 흐름을 포함합니다. `App.test.tsx`는 카드 추가 후 복습 완료와 세션 복원 두 흐름을 다룹니다.

## 현재 강점

- `StudyRepository` 추상화가 있어 저장소 구현 교체 여지가 남아 있습니다.
- 로컬 우선 저장 구조와 세션 복원이 이미 들어 있어 MVP 사용성은 충분히 확보되어 있습니다.
- Gemini 프록시가 클라이언트 키 노출을 막는 구조라 현재 개발 환경 기준으로 안전합니다.
- SRS 계산, 입력 검증, AI 자동 채우기, 핵심 사용자 흐름에 대한 최소 테스트가 이미 마련되어 있습니다.

## 확인된 개선 필요

### 1. 삭제 기능 부재

- 현재는 잘못 추가한 카드를 정리할 수 없습니다.
- 오타나 중복 입력이 누적되면 단어장 관리가 빠르게 불편해질 수 있습니다.
- 이는 실제 사용 중 바로 체감되는 데이터 관리 문제이므로 최우선 개선 후보입니다.
- 권장 확장안: repository와 UI에 삭제 기능을 추가하고, `entries`와 `reviews`를 함께 정리합니다.

### 2. 백업은 가능하지만 복원 범위가 닫혀 있지 않음

- 현재 `exportData()`는 `entries`와 `reviews`만 내보냅니다.
- 반면 진행 중 세션과 마지막 요약은 `localStorage`에 따로 저장됩니다.
- 따라서 import를 단순히 추가하면 카드와 복습 상태는 옮겨도 “이어 하기” 세션과 마지막 요약은 복원되지 않는 반쪽 기능이 됩니다.
- 우선순위는 import 버튼 추가 자체보다 “무엇을 백업/복원 대상으로 볼지”를 먼저 결정하는 것입니다.
- 권장 확장안: 제안 API로 `importData(...)`를 도입하되, 세션/요약 포함 여부 또는 import 시 초기화 정책을 함께 정의합니다.

### 3. 정적 프로덕션 배포에서 AI 자동 채우기 불가

- Gemini 프록시는 Vite의 `configureServer`와 `configurePreviewServer`에만 연결되어 있습니다.
- 따라서 정적 호스팅만으로 배포하면 `/api/spanish-word-autofill` 엔드포인트가 사라집니다.
- 외부 공개 배포 전에는 서버리스 프록시나 별도 backend 등 production runtime을 먼저 정해야 합니다.

### 4. 접근성 보강 여지

- 복습 카드는 네이티브 `<button>`이라 키보드 포커스 자체는 가능하지만, 뒤집힌 상태를 전달하는 `aria-expanded` 같은 상태 정보는 없습니다.
- rating 버튼도 현재는 영문 레이블 중심이라 한국어 설명이 보강되면 이해가 더 쉬워집니다.
- 이는 치명적 버그라기보다 빠르게 개선 가능한 품질 항목입니다.

### 5. 테스트 보강 여지

- 현재 최소 핵심 테스트는 있으나 `stats.ts`, `format.ts`, `storage.ts`, `sessionStorage.ts`는 직접 단위 테스트가 없습니다.
- 저장/복원 기능이 확장되기 전에 이 영역을 먼저 보강하는 편이 안전합니다.

## 선택적 확장 아이디어

- 홈 탭 추가는 UX 선택지입니다. 현재도 상단 브랜드 링크로 홈 복귀가 가능하므로 핵심 결함으로 보기는 어렵습니다.
- 단어장 정렬/필터, 복습 단축키, 카드 플립 애니메이션은 사용성 향상 항목으로 다음 단계 후보입니다.
- PWA, 다크 모드, 통계 시각화, Context 리팩터링, IndexedDB 인덱스 최적화는 현 시점에서는 후순위 확장에 가깝습니다.
- `createEntry` 중복 검사 최적화는 데이터 규모가 커질수록 의미가 커지므로, 현재 단계에서는 중간 이하 우선순위로 보는 편이 적절합니다.

## 권장 실행 순서

1. 삭제 기능 추가
2. 백업/복원 범위 재설계 후 import 도입
3. 프로덕션용 AI 프록시 구조 결정
4. 저장/복원 및 접근성 보강
5. 사용성 확장과 후순위 기능 검토
