# Objective

Turn a feature request into a clear product slice that is worth building now.

## Process

1. Restate the proposal in user-facing terms.
2. Identify the primary user, pain point, and expected outcome.
3. Define the smallest shippable slice and explicit non-goals.
4. Evaluate sequencing, dependencies, and adoption risk.
5. Recommend go, trim, split, defer, or reject with a reason.

## Output

Produce:

- User value summary
- In-scope and out-of-scope definition
- Suggested release slice
- Key product risks or dependencies
- Success signal or metric
- Recommendation with tradeoffs

## Guardrails

- Prefer the smallest lovable slice over a broad launch.
- Separate must-haves from follow-up ideas.
- Make missing information explicit instead of hiding it.
